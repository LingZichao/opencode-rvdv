# Case 018: 物理页分配碎片化与映射复用压力（新方向）

请生成一个 RV64 单线程 ISG 脚本，目标是在分页场景下模拟页分配碎片化，构造多段地址区间并验证映射复用与失败候选。

## 任务目标

该 case 关注页分配与地址映射策略层面的细节，不仅是普通 VA 生成；重点验证碎片化下的可用页与不可用页切换行为。

## 量化要求

1. 指令总数 120 到 200 条。
2. 地址分配/映射相关操作不少于 26 条。
3. 至少构造 4 个地址区间，其中 2 个作为碎片化小区间。
4. 至少 3 次成功映射复用访问。
5. 至少 2 次分配失败或受限候选访问。
6. 访存指令不少于 24 条。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 阶段 A：页池与区间准备。
2. 阶段 B：大区间成功映射访问。
3. 阶段 C：碎片区间分配与复用访问。
4. 阶段 D：受限/失败候选区间访问。
5. 阶段 E：汇总验证与退出。

## 数据编排要求

1. 维护 `RANGE_SET = {R_BIG0, R_BIG1, R_FRAG0, R_FRAG1}`。
2. 每个区间至少两个偏移样本。
3. 至少一组访问模板在 `R_BIG*` 与 `R_FRAG*` 中复用，隔离“分配拓扑”因素。
4. 在失败候选区间前后写阶段签名，便于 trace 标注。

## 通过判定

1. trace 可区分大区间与碎片区间行为。
2. 成功复用和失败候选都可统计。
3. 脚本最终正常退出。

## 实现边界

- 仅限 RV64 单线程。
- 不引入向量主线。
- 重点是页分配碎片化与映射复用细节。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释区间碎片化、映射复用与失败候选如何满足指标。
