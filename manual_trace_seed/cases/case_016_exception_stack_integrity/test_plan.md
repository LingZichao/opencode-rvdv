# Case 016: 异常处理栈保存/恢复一致性（新方向）

请生成一个 RV64 单线程 ISG 脚本，目标是在多轮异常进入/返回过程中验证处理栈保存与恢复的一致性。

## 任务目标

该 case 关注异常处理器栈语义，而非仅触发异常：需要证明进入异常后关键上下文被保存，返回后可恢复并继续执行。

## 量化要求

1. 指令总数 120 到 200 条。
2. 异常事件不少于 4 次，至少覆盖 2 类异常来源。
3. 每次异常往返至少校验 1 组保存寄存器和 1 组恢复寄存器。
4. 栈相关内存访问不少于 12 条。
5. 至少 3 次返回后继续执行并完成后继验证块。
6. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 阶段 A：处理栈基线与标记寄存器准备。
2. 阶段 B：第一轮异常进入/返回与栈一致性检查。
3. 阶段 C：第二轮异常进入/返回（不同触发类型）与检查。
4. 阶段 D：跨轮保存恢复一致性汇总检查。
5. 阶段 E：收敛退出。

## 数据编排要求

1. 维护 `SAVE_SET`、`RESTORE_SET` 两组寄存器簇。
2. 每轮异常前后写路径签名寄存器，确保 trace 可分段。
3. 至少一轮在保存字段不变条件下仅变化触发类型，隔离“处理栈语义”影响。
4. 栈窗口地址应固定在小范围内，便于跟踪重复访问。

## 通过判定

1. trace 可定位异常进入/返回边界。
2. 保存与恢复寄存器检查结果一致且可追踪。
3. 脚本最终正常退出。

## 实现边界

- 仅限 RV64 单线程。
- 不引入向量主线。
- 重点是异常处理栈一致性验证。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释保存/恢复验证链路如何满足指标。
