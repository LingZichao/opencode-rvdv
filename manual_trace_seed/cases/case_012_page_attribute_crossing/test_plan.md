# Case 012: 页属性跨窗口与跨页边界异常（超高难）

请生成一个 RV64 单线程 ISG 脚本，目标是在相邻页窗口中构造属性差异、跨页访问和受限访问对照。

## 任务目标

验证 agent 是否能同时处理页大小、跨页边界、权限属性和地址复用，而不是只做普通 load/store。

## 量化要求

1. 指令总数 120 到 210 条。
2. 访存指令不少于 30 条，load/store 各不少于 12 条。
3. 至少构造 3 个页属性窗口：RW、RO 或 NX、以及无效/受限窗口。
4. 至少 2 次跨页边界访问。
5. 至少 2 次成功访问与 2 次受限访问候选。
6. 地址生成与页配置相关指令不少于 20 条。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 阶段 A：页窗口准备与属性配置。
2. 阶段 B：窗口内成功访问基线。
3. 阶段 C：跨页边界访问。
4. 阶段 D：受限页窗口访问对照。
5. 阶段 E：结果收敛与退出。

## 数据编排要求

1. 维护 `PAGE_WIN_SET = {WIN_RW, WIN_LIMIT, WIN_HOLE}`。
2. 每个窗口至少 2 个偏移样本，覆盖页内和跨页边界位置。
3. 至少一组访问采用同一地址生成模板，仅更改目标窗口属性。
4. 成功窗口和受限窗口之间要有阶段签名寄存器区分。

## 通过判定

1. trace 可区分三类页窗口行为。
2. 跨页访问与受限访问均可识别。
3. 主线收敛且正常退出。

## 实现边界

- 仅限 RV64 单线程。
- 不引入向量主线。
- 重点是页属性细节与跨页访问语义。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释页窗口、跨页访问和受限对照如何满足指标。
