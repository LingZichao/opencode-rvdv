# Case 001: 窄窗口寄存器 RAW 依赖

请生成一个 FORCE-RISCV ISG 脚本，目标是在 RV64 单线程场景下制造一个可控的窄窗口整数寄存器 RAW 依赖序列。

## 任务目标

生成一个 RV64 单线程脚本，围绕一个指定 GPR 构造可控 RAW 依赖窗口。该 case 的重点不是复杂功能，而是让脚本表现出清晰、可观测的“预热阶段 + 依赖突发阶段”结构。

## 结构要求

1. 脚本至少分成两个阶段：
   - 预热阶段：生成若干随机或半随机整数 ALU 指令。
   - 依赖阶段：生成显式 writer-reader 指令对，使后一条指令读取前一条指令写回的寄存器。
2. 预热阶段至少包含 6 条整数 ALU 指令。
3. 依赖阶段至少包含 8 组 writer-reader 指令对。
4. 至少一半的依赖对围绕同一个指定 GPR 展开。
5. 依赖关系应尽量集中在较近的指令窗口内，而不是被大段无关指令打散。
6. 结束前写 `x10/a0 = 0`，然后发出 `M5EXIT##RISCV`。

## 场景约束

1. 选择一个非 `x0` 的 GPR 作为核心依赖寄存器。
2. 在依赖阶段开始前，这个寄存器需要具有一个已知的非零值。
3. 预热阶段应尽量避免该寄存器被随机整数流误用。
4. 进入依赖阶段后，该寄存器必须重新参与 writer-reader 指令对构造。
5. 依赖阶段里，除核心依赖寄存器外，也可以混入少量其他寄存器依赖对，但不应稀释主场景。

## 实现边界

- 只使用整数 ALU 类指令即可，不需要 load/store、异常、分页或 privilege 逻辑。
- 不要引入多线程、向量或浮点序列。
- 实现方式由 generator 自行决定，只要最终行为满足上述场景约束即可。
- 脚本应保持可编译、结构清晰、注释简洁。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释脚本如何满足上面的阶段结构和依赖行为约束。
