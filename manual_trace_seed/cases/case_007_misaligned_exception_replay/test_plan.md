# Case 007: 非对齐访存异常与恢复重放（高难度）

请生成一个 RV64 单线程 ISG 脚本，目标是在受控地址窗口内交替构造对齐访存与非对齐访存，触发可观测异常路径，并在异常后继续执行并完成收敛退出。

## 任务目标

形成“对齐基线 -> 非对齐触发 -> 恢复重放 -> 收敛退出”的完整链路，重点观察访存对齐约束、异常边界与后续控制流连续性。

## 量化要求

1. 脚本总指令数控制在 100 到 180 条。
2. 内存访问指令不少于 28 条，其中：
   - 对齐访存不少于 12 条
   - 非对齐候选访存不少于 8 条
3. 至少包含 3 种访存宽度组合（例如 byte/half/word/double 的任意 3 种）。
4. 至少触发 2 次非对齐异常候选事件，且至少 1 次可在 trace 中明确识别为异常路径。
5. 至少包含 2 轮“同地址族”访问：
   - 一轮对齐访问成功
   - 一轮非对齐偏移导致异常或异常候选
6. 控制流引导指令（branch/jump）不少于 8 条，用于界定异常前后边界与重放路径。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 脚本至少分 5 个阶段：
   - 阶段 A：地址窗口建立与对齐基线写入
   - 阶段 B：对齐读取验证
   - 阶段 C：非对齐偏移构造与异常触发
   - 阶段 D：异常后恢复与重放访问
   - 阶段 E：收敛退出
2. 阶段 C 的非对齐访问必须由显式偏移构造产生，而不是随机地址碰撞。
3. 阶段 D 需复用阶段 A 的基址或同族偏移，体现“同场景重放”而非全新场景。
4. 阶段 C 与阶段 D 之间应有可定位的控制流边界（分支或跳转块）。

## 数据编排要求

1. 定义一个主窗口基址 BASE，以及至少 2 组偏移族：
   - ALIGNED_SET：满足对应访存宽度对齐
   - MISALIGNED_SET：在同一宽度下有意破坏对齐
2. 对齐路径先执行写入后读回，形成可验证基线。
3. 非对齐路径至少覆盖两种宽度的 misaligned 偏移样本。
4. 恢复阶段对同地址族再次执行访问，证明脚本仍在可控上下文中。
5. 在关键窗口前后插入少量整数比较或标记指令，帮助 trace 划分阶段。

## 通过判定

1. trace 中可区分对齐成功窗口与非对齐异常窗口。
2. 至少 1 次非对齐事件在行为上表现出异常路径特征。
3. 异常后存在明确恢复块并继续执行到正常退出。
4. 同一地址族在异常前后被重复使用，体现重放而非随机漂移。

## 实现边界

- 仅限 RV64 单线程。
- 不引入多核、向量或浮点主线。
- 重点是访存对齐语义与异常恢复可观测性，不要求实现完整操作系统异常框架。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释“对齐基线 -> 非对齐触发 -> 恢复重放 -> 收敛退出”如何满足量化指标。
