# Case 004: 浮点约束数据混合（量化增强版）

请生成一个 FORCE-RISCV ISG 脚本，目标是在 RV64 场景下构造可追踪的受约束浮点输入池，并在多精度乘除链中重复复用这些输入。

## 任务目标

构建“受约束输入准备 -> 多精度乘除复用 -> 结果收敛退出”的线性浮点路径，让 trace 中可以区分边界值输入与常规受约束输入的作用。

## 量化要求

1. 浮点相关指令总数不少于 24 条。
2. 乘除类指令不少于 12 条，且必须包含：
   - 单精度 `fmul.s` 与 `fdiv.s`
   - 双精度 `fmul.d` 与 `fdiv.d`
3. 至少准备 6 个可复用浮点源操作数寄存器（例如 f2-f7）。
4. 至少 70% 的乘除指令应复用上述固定操作数池，禁止每条指令都重新造数。
5. 至少 4 条指令使用“边界敏感输入”（如接近 0、接近最大有限值、符号翻转对）。
6. 脚本全程保持线性执行，不出现条件分支或循环。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 阶段 A（数据准备）：
   - 使用整数到浮点搬运/转换与内存装载中的一种或组合，建立受约束浮点池。
   - 至少包含一组“窄格式或边界值”输入和一组“常规受约束”输入。
2. 阶段 B（单精度链）：
   - 以同一批寄存器为主，执行多条 `fmul.s`/`fdiv.s`。
   - 至少形成 2 条长度 >= 3 的 RAW 依赖链。
3. 阶段 C（双精度链）：
   - 复用阶段 A 的数据语义，执行 `fmul.d`/`fdiv.d`。
   - 至少形成 2 条长度 >= 3 的 RAW 依赖链。

## 数据编排要求

1. 输入池至少覆盖以下类别中的 4 类：
   - 正常小值（接近 0 但非 0）
   - 正常中值（如 1.0、2.0、0.5）
   - 大幅值（接近上界但避免溢出到 Inf）
   - 符号成对值（+x / -x）
   - 精度敏感值（可触发舍入差异）
2. 明确指定“边界敏感子集”寄存器（例如 f2/f3），并在单精度和双精度阶段都至少使用一次。
3. 在乘除序列中插入少量非乘除浮点指令（如 fadd/fsub/fsgnj）用于数据重组，但不得改变“乘除主线”占比。

## 通过判定

1. trace 中可统计出 >=24 条浮点指令，且 `fmul/fdiv` 覆盖 `s` 与 `d` 两种精度。
2. 可识别固定操作数池被重复复用（复用率 >=70%）。
3. 边界敏感输入在两种精度路径中都被实际消费。
4. 指令流线性，无分支跳转干扰。

## 实现边界

- 仅限 RV64 单线程。
- 不将异常、分页、多核、向量作为主场景。
- 重点是浮点输入约束与复用行为，不要求复杂数学功能正确性证明。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释输入池如何构造、如何在 `fmul/fdiv` 中复用、以及如何满足量化指标。
