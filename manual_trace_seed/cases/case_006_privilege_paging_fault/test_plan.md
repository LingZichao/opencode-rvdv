# Case 006: 特权切换与分页故障链路（高难度）

请生成一个 RV64 单线程 ISG 脚本，目标是在同一条执行主线中构造“特权级切换 + 页表映射 + 页故障触发 + 故障后收敛退出”的可观测行为。

## 任务目标

构建一个对微架构控制路径和地址翻译路径都施压的复杂场景，使 trace 能明确定位：

- 特权状态切换前后的访存行为差异
- 合法页访问与故障页访问的边界
- 故障发生后如何回到可控执行路径并退出

## 量化要求

1. 脚本总指令数控制在 90 到 160 条。
2. 系统状态/CSR 相关指令不少于 18 条。
3. 内存访问指令不少于 24 条，其中 load/store 各不少于 10 条。
4. 地址翻译相关行为需覆盖至少 3 类地址：
   - 显式映射且可访问地址
   - 映射属性受限地址（权限不足）
   - 未映射地址（用于触发页故障）
5. 至少出现 2 次特权相关状态切换（例如 M->S，再回到受控状态）。
6. 至少出现 2 次“可成功提交”的访存和 1 次“应触发页故障”的访存。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 脚本至少分 4 个阶段：
   - 阶段 A：基线状态建立（CSR 与基本地址上下文）
   - 阶段 B：合法映射访问窗口（成功 load/store）
   - 阶段 C：故障触发窗口（权限或映射缺失导致 fault）
   - 阶段 D：故障后收敛与退出
2. 阶段 B 与阶段 C 必须共享部分地址生成链路，避免完全独立的两套脚本。
3. 至少一条故障触发访问应与最近一次成功访问只差页属性或页表有效性，形成对比样本。
4. 阶段 D 中必须存在明确的“后故障可继续执行”证据块（非空指令块）。

## 数据编排要求

1. 准备三组虚拟地址集合：
   - VA_OK：可读可写
   - VA_PROT：权限受限
   - VA_HOLE：无有效映射
2. 每组至少包含 2 个偏移样本，且地址间距可区分页粒度与行内偏移效应。
3. 对 VA_OK 先执行初始化写入，再执行读回校验，形成基线正确性。
4. 对 VA_PROT 或 VA_HOLE 执行至少一次访问触发故障，并保留前后状态可观测差异。
5. 在故障前后都保留少量整数指令，用于界定故障窗口边界。

## 通过判定

1. trace 中可定位特权/CSR 配置阶段与后续访存阶段的分界。
2. trace 中可识别至少 2 次成功访存与至少 1 次页故障触发事件。
3. 故障事件前后存在可追踪的上下文变化，且脚本最终可正常退出。
4. 行为主线集中在特权和分页语义，不被无关随机指令淹没。

## 实现边界

- 仅限 RV64 单线程。
- 不引入多核、向量或浮点主线。
- 重点是特权状态与地址翻译行为的可观测性，不要求实现通用 OS 场景。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，按“状态建立 -> 合法访问 -> 故障触发 -> 收敛退出”解释脚本如何满足量化指标。
