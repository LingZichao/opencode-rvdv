# Case 010: 向量掩码访存与地址属性耦合（高难度）

请生成一个 RV64 单线程 ISG 脚本，目标是在向量访存路径中组合“掩码使能 + 索引寻址 + 地址属性差异 + 异常候选窗口”，形成高维行为耦合场景。

## 任务目标

该 case 重点是把向量数据通路与地址约束通路耦合，要求同一批向量访存在不同掩码与地址属性条件下表现出可区分行为。

## 量化要求

1. 脚本总指令数控制在 120 到 210 条。
2. 向量相关指令不少于 30 条。
3. 向量访存指令不少于 14 条，至少覆盖：
   - unit-stride 或 strided 访问
   - indexed 访问
   - masked 访问
4. 至少 3 次 vset* 配置切换（例如不同元素宽度/LMUL/有效长度组合）。
5. 至少 2 组地址属性对照窗口：
   - 一组可成功访问
   - 一组异常候选或受限访问
6. 至少 2 次“同一向量索引模式 + 不同掩码”的重放对照。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 至少分 6 个阶段：
   - 阶段 A：向量配置与索引/掩码寄存器准备
   - 阶段 B：可访问地址窗口访存
   - 阶段 C：掩码变体重放
   - 阶段 D：受限地址属性窗口访存
   - 阶段 E：结果/状态读回标记
   - 阶段 F：收敛退出
2. 阶段 B 与阶段 D 必须复用同类访存模板，主要差异应来自地址属性与掩码配置。
3. 阶段 C 至少包含一次“只改掩码不改索引”的重放，用于隔离掩码效应。
4. 若阶段 D 出现异常候选，后续必须有可观测的继续执行块。

## 数据编排要求

1. 准备两组地址窗口：
   - VADDR_OK：可稳定访问
   - VADDR_LIMIT：权限或映射属性受限
2. 准备至少两组掩码模式：
   - MASK_DENSE：高占空比
   - MASK_SPARSE：低占空比
3. 准备至少一组索引向量，且在两个地址窗口中复用。
4. 每轮访存后写入路径签名或状态标记，便于 trace 侧区分阶段。
5. 掩码、索引、地址属性三类变化应避免同一时刻全变，至少保留一组单变量对照。

## 通过判定

1. trace 中可分辨向量配置切换、掩码重放和地址属性对照窗口。
2. 可统计出至少两轮“同索引不同掩码”的行为对照。
3. 可观测到受限地址窗口与可访问窗口的行为差异。
4. 脚本最终正常退出。

## 实现边界

- 仅限 RV64 单线程。
- 主线聚焦向量访存与地址属性耦合，不扩展到多核并发。
- 重点是行为可观测性与对照设计，不要求数值算法复杂度。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释向量配置、掩码重放和地址属性对照如何满足量化指标。
