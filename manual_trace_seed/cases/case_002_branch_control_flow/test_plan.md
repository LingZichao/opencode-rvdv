# Case 002: 控制流双路径（量化增强版）

请生成一个 RV64 单线程 ISG 脚本，目标是在同一段短程序中稳定呈现：

- 条件分支命中路径（taken）
- 条件分支顺序路径（not-taken）
- 寄存器间接跳转路径（jalr）

重点是微架构可观测的控制流行为，而不是 API 调用方式。

## 任务目标

构造一个具有明确基本块边界的控制流场景，使 trace 中可以直接定位分支决策点、落点和回流点，并且同一脚本内同时覆盖 taken / not-taken / indirect jump 三种行为。

## 量化要求

1. 指令总量控制在 40 到 80 条（不含注释）。
2. 控制流前预热区至少 8 条整数 ALU 指令（例如 addi/add/xor/and/slli）。
3. 条件分支指令至少 6 条，且至少覆盖 2 种分支类型（例如 beq/bne/blt/bge）。
4. 分支结果要求：
   - 至少 2 次 taken
   - 至少 2 次 not-taken
5. 寄存器间接跳转（jalr）至少 2 次，且跳转目标来自显式构造的目标寄存器。
6. 控制流相关指令（branch/jal/jalr）占比不低于 20%。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 至少划分 4 个可识别代码块：
   - warmup
   - branch_decision
   - branch_target / fallthrough
   - indirect_jump_target
2. 条件分支必须通过确定性寄存器值驱动，而不是随机概率触发。
3. jalr 的目标地址必须在跳转前由地址构造指令建立（例如 auipc + addi，或已知标签地址装入寄存器）。
4. taken 与 not-taken 两条路径都必须包含至少 3 条非空操作指令，禁止空块。
5. 跳转后必须有汇合点（join block），用于继续执行并收尾。

## 数据编排要求

1. 在预热区准备两组比较操作数：
   - 组 A：用于触发 taken（例如 rs1 == rs2）
   - 组 B：用于触发 not-taken（例如 rs1 != rs2）
2. 所有参与分支比较的寄存器在分支前必须被显式写入，禁止依赖未定义初值。
3. 间接跳转目标寄存器至少准备两次不同目标值，形成 2 次可区分的 jalr 行为。
4. 路径内允许少量整数运算扰动，但不得覆盖控制流关键寄存器（分支比较寄存器和跳转目标寄存器）。

## 通过判定

1. 在 trace 中可定位至少 2 次 taken、2 次 not-taken 的条件分支。
2. 在 trace 中可定位至少 2 次 jalr，且目标 PC 属于预设目标块。
3. 所有路径最终汇合并正常退出，无死循环和无目标跳转。

## 实现边界

- 仅限 RV64 单线程。
- 不引入浮点、向量、多核、异常或分页主线。
- 重点是控制流结构可读性和可验证性，不要求复杂算法。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，按“预热 -> 分支双路径 -> 间接跳转 -> 汇合退出”解释如何满足上述量化指标。
