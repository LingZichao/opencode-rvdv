# Case 017: 压缩指令控制流对齐与回汇验证（新方向）

请生成一个 RV64 单线程 ISG 脚本，目标是在包含压缩指令（C 扩展）的短窗口中验证分支目标对齐、跳转回汇和异常候选边界。

## 任务目标

该 case 关注压缩指令长度与控制流目标对齐细节，验证 agent 能区分 16-bit/32-bit 指令混合下的路径落点语义。

## 量化要求

1. 指令总数 100 到 170 条。
2. 压缩指令不少于 20 条。
3. 控制流指令不少于 18 条。
4. 至少 2 次压缩-非压缩混合块的分支进入与回汇。
5. 至少 2 次寄存器间接跳转（jalr）目标由显式地址链构造。
6. 至少 1 处对齐异常候选窗口（可作为候选，不强制实际异常）。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 阶段 A：混合指令流预热（16-bit/32-bit）。
2. 阶段 B：压缩块分支与落点验证。
3. 阶段 C：jalr 目标切换与回汇。
4. 阶段 D：对齐候选窗口与恢复块。
5. 阶段 E：收敛退出。

## 数据编排要求

1. 维护 `C_BLOCK_SET` 与 `MIX_BLOCK_SET` 两类代码块签名。
2. 分支目标至少两组：`TGT_ALIGN_OK` 与 `TGT_ALIGN_CANDIDATE`。
3. 至少一组场景固定分支模板，仅改变块内指令长度分布。
4. 每个回汇点写路径签名寄存器，便于 trace 对齐。

## 通过判定

1. trace 可区分压缩混合块与普通块。
2. 控制流落点和回汇点可统计。
3. 对齐候选窗口可定位，最终可正常退出。

## 实现边界

- 仅限 RV64 单线程。
- 不引入向量主线。
- 重点是压缩指令与控制流对齐细节。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释压缩混合块与控制流对齐验证设计。
