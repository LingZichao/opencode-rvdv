# Case 009: 推测分支与回汇重收敛（高难度）

请生成一个 RV64 单线程 ISG 脚本，目标是在紧凑窗口内构造“推测分支扰动 -> 多路径展开 -> 回汇重收敛 -> 间接跳转再收敛”的双层控制流压力场景。

## 任务目标

该 case 重点不是分支数量，而是路径拓扑与收敛结构的可观测复杂性，要求同一段程序里出现可区分的一级分叉和二级分叉。

## 量化要求

1. 脚本总指令数控制在 100 到 170 条。
2. 控制流指令（branch/jal/jalr）不少于 24 条。
3. 条件分支至少 10 条，且 taken/not-taken 各不少于 4 次。
4. 至少 2 处回汇点（join block）被多个前驱路径共同到达。
5. 至少 2 次 jalr 间接跳转，且跳转目标由寄存器构造链显式给出。
6. 至少 1 处 loop-like 回边，但总执行必须可终止。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 至少分 6 个阶段：
   - 阶段 A：预热与比较数据准备
   - 阶段 B：一级分叉（主条件分支）
   - 阶段 C：二级分叉（路径内再分支）
   - 阶段 D：一级回汇
   - 阶段 E：jalr 目标跳转与二级回汇
   - 阶段 F：收敛退出
2. 一级和二级分叉应共享部分状态寄存器，制造路径间语义耦合。
3. 至少一处回汇后必须继续执行依赖前序路径结果的比较或标记操作。
4. loop-like 回边必须有明确退出门限，禁止不可控循环。

## 数据编排要求

1. 准备两组比较寄存器簇：
   - CMP_A：驱动一级分叉
   - CMP_B：驱动二级分叉
2. 准备两组目标地址寄存器簇：
   - JT_A：一级 jalr 目标
   - JT_B：二级 jalr 目标
3. 在进入每个回汇点前，至少写入一个路径签名寄存器，用于 trace 中识别路径来源。
4. 允许插入少量整数算术扰动，但不得破坏分支关键寄存器与跳转目标寄存器。

## 通过判定

1. trace 中可定位一级/二级分叉与对应回汇点。
2. taken/not-taken、jalr 目标和回边行为均可被统计。
3. 最终路径收敛并正常退出。
4. 路径签名寄存器可区分至少两条不同来源路径。

## 实现边界

- 仅限 RV64 单线程。
- 不把浮点、向量作为主线。
- 重点是控制流拓扑复杂度与重收敛可观测性。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释双层分叉、两级回汇和终止控制如何满足量化指标。
