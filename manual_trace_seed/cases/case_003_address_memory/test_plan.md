# Case 003: 地址窗口复用（量化增强版）

请生成一个 RV64 单线程 ISG 脚本，目标是在一个小地址窗口内构造可重复、可验证的读写访问模式，突出“初始化 -> 读取验证 -> 再次复用”。

## 任务目标

让 trace 能清楚反映同一批偏移地址被多轮访问，而不是随机分散访问；并且至少有一轮读操作明确依赖于前序写入结果。

## 量化要求

1. 脚本总指令数 45 到 90 条。
2. 地址窗口大小固定为 64B 或 128B（二选一，需在说明中声明）。
3. 内存访问总数不少于 20 条，其中：
	- store 不少于 8 条
	- load 不少于 8 条
4. 至少 75% 的内存访问落在同一地址窗口内。
5. 至少 6 个不同偏移被复用（每个偏移至少被访问 2 次）。
6. 至少 4 组严格的 store->load 配对（load 读取先前同地址 store 的结果）。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 脚本分 3 个阶段：
	- 阶段 A（窗口建立）：基址生成 + 对齐 + 初始写入
	- 阶段 B（验证读取）：对阶段 A 写入地址执行读取
	- 阶段 C（复用扰动）：在同窗口内二次写入与再次读取
2. 地址生成应包含显式基址寄存器和偏移寄存器运算（例如 addi/slli/add）。
3. 访存指令应覆盖至少 2 种粒度（例如 sw+lw 或 sd+ld）。
4. 阶段 B 和阶段 C 中禁止重新分配全新窗口基址，必须复用阶段 A 的窗口。

## 数据编排要求

1. 先定义固定偏移集合，例如 {0, 8, 16, 24, 32, 40, 48, 56}（可根据窗口大小调整）。
2. 阶段 A 对偏移集合写入可区分数据模式（例如递增模式、异或模式、常量+索引）。
3. 阶段 B 按相同偏移集合读取，并用整数比较指令验证读回数据一致性。
4. 阶段 C 对其中至少一半偏移覆写新模式，再读取确认已更新。
5. 可加入少量窗口外访问作为干扰，但总量不得超过全部访存的 25%。

## 通过判定

1. trace 中可识别固定窗口基址和重复偏移访问。
2. 至少 4 组 store->load 存在明确前后关系，且读取值匹配预期模式。
3. 可区分“初始化值”和“复写值”两轮数据生命周期。
4. 脚本正常退出，无越界异常或非预期跳转。

## 实现边界

- 仅限 RV64 单线程。
- 不引入浮点、向量、多核或异常处理主线。
- 重点是地址窗口复用和数据可追踪性，不追求复杂算法。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释窗口大小选择、偏移编排、以及 store->load 复用如何满足量化指标。
