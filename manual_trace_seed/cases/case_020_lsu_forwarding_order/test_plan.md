# Case 020: LSU Store-to-Load Forwarding 与顺序窗口专项（微架构功能）

请生成一个 RV64 单线程 ISG 脚本，目标是在小地址窗口内专项验证 store-to-load forwarding、部分重叠访问与内存顺序边界行为。

## 任务目标

该 case 关注 LSU 微架构行为：同址/邻址/部分重叠访问下，读取应来自前序 store 还是内存基值；并要求出现可观测顺序窗口与收敛验证。

## 量化要求

1. 指令总数 120 到 200 条。
2. 访存指令不少于 36 条，load/store 各不少于 14 条。
3. 至少 8 组严格 store->load 近邻配对。
4. 至少 4 组部分重叠访问（不同宽度或不同偏移）。
5. 至少 3 组同址重放访问（同模板重复）。
6. 至少 2 个顺序窗口边界块（用于区分前后访问组）。
7. 至少 1 组“基线内存值”与“前递值”对照读取。
8. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 阶段 A：地址窗口与基线内存初始化。
2. 阶段 B：同址 store->load 前递窗口。
3. 阶段 C：部分重叠访问窗口（宽度/偏移变化）。
4. 阶段 D：顺序边界切换与重放验证。
5. 阶段 E：收敛退出。

## 数据编排要求

1. 定义 `BASE_WIN` 与偏移族 `OFF_EXACT`、`OFF_OVERLAP`。
2. 对每组 store 写入可区分模式值，随后紧邻 load 读取。
3. 至少一组测试仅变化访问宽度（保持地址族不变）。
4. 至少一组测试仅变化偏移（保持宽度不变）。
5. 每个顺序窗口前后写路径签名寄存器，便于 trace 标注。

## 通过判定

1. trace 可统计同址配对、重叠访问和重放窗口。
2. 可区分基线值读取与前递值读取。
3. 脚本最终正常退出。

## 实现边界

- 仅限 RV64 单线程。
- 不引入向量主线。
- 重点是 LSU 前递与顺序窗口行为可观测性。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释前递配对、重叠访问和顺序窗口如何满足指标。
