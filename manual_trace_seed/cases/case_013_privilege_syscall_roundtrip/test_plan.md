# Case 013: 系统调用往返与跨级状态一致性（超高难）

请生成一个 RV64 单线程 ISG 脚本，目标是通过多次系统调用触发跨特权级往返，并验证返回后状态一致性与地址上下文连续性。

## 任务目标

强调 syscall 触发、返回路径、CSR 状态与访存上下文之间的耦合验证。

## 量化要求

1. 指令总数 120 到 200 条。
2. 至少 4 次系统调用类异常触发。
3. 至少 3 次成功返回到预期后继块。
4. CSR/特权状态相关指令不少于 22 条。
5. 返回后访存或控制流验证块不少于 3 个。
6. 至少 2 次在不同上下文下复用同一地址族或寄存器簇。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 阶段 A：特权/系统调用基线准备。
2. 阶段 B：首轮 syscall 触发与返回。
3. 阶段 C：第二轮 syscall（不同上下文参数）与返回。
4. 阶段 D：跨轮一致性验证（状态与路径签名）。
5. 阶段 E：收敛退出。

## 数据编排要求

1. 定义 `SCALL_SITE_SET` 至少 4 个触发点，覆盖两种上下文配置。
2. 每轮 syscall 前后都写入路径签名寄存器。
3. 维护 `STATE_CHECK_SET`，至少包含 1 组 CSR 检查与 1 组普通寄存器检查。
4. 在两轮之间保留短普通块避免事件连粘。

## 通过判定

1. trace 中 syscall 触发与返回链可分段。
2. 返回后状态检查满足一致性预期。
3. 最终正常退出。

## 实现边界

- 仅限 RV64 单线程。
- 不引入向量主线。
- 重点是 syscall 往返和状态连续性。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释多轮 syscall 往返与一致性验证设计。
