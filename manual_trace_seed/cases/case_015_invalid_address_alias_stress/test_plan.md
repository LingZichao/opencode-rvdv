# Case 015: 无效地址与别名映射复用压力（超高难）

请生成一个 RV64 单线程 ISG 脚本，目标是在同一地址族内混合有效映射、别名映射和无效地址访问，并验证不同路径的可区分行为。

## 任务目标

强调地址验证、地址别名与异常候选之间的细粒度对照，避免把所有失败都归因于随机无效地址。

## 量化要求

1. 指令总数 120 到 205 条。
2. 地址生成/验证相关操作不少于 24 条。
3. 访存指令不少于 28 条。
4. 至少 3 类地址族：VALID、ALIAS、INVALID。
5. 至少 2 次别名地址复用访问。
6. 至少 2 次无效地址访问候选。
7. 至少 3 次成功访问（VALID 或 ALIAS）。
8. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 阶段 A：地址族准备与标记。
2. 阶段 B：VALID 窗口成功基线。
3. 阶段 C：ALIAS 窗口复用对照。
4. 阶段 D：INVALID 窗口异常候选。
5. 阶段 E：收敛验证与退出。

## 数据编排要求

1. 定义 `ADDR_FAMILY = {VA_VALID, VA_ALIAS, VA_INVALID}`。
2. 至少一组 VALID/ALIAS 使用相同数据模板，隔离映射差异。
3. INVALID 组至少包含两种偏移样本，避免单点偶然。
4. 每个阶段写路径签名寄存器，支持 trace 分段。
5. 至少一组 ALIAS 访问在 INVALID 窗口前后各执行一次，形成对照链。

## 通过判定

1. trace 可区分 VALID/ALIAS/INVALID 三类窗口。
2. ALIAS 复用行为可识别且与 VALID 基线形成对照。
3. INVALID 候选事件可见，且执行最终收敛退出。

## 实现边界

- 仅限 RV64 单线程。
- 不引入向量主线。
- 重点是地址验证、别名映射与无效访问对照。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释三类地址族和对照链路如何满足量化指标。
