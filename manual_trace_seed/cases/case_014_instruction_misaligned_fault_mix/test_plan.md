# Case 014: 指令非对齐与分支取指故障混合（超高难）

请生成一个 RV64 单线程 ISG 脚本，目标是在控制流压力下混合触发指令非对齐异常候选与取指访问故障候选，并验证恢复路径。

## 任务目标

构造取指路径相关的冷门场景，区分“分支目标构造错误导致的非对齐候选”和“目标页属性导致的取指故障候选”。

## 量化要求

1. 指令总数 130 到 210 条。
2. 控制流指令不少于 26 条。
3. 指令非对齐候选事件不少于 2 次。
4. 取指访问故障候选事件不少于 2 次。
5. 至少 3 次异常后可恢复到主线锚点。
6. 至少 2 个回汇点，至少 1 条受控回边。
7. 结束前写 x10/a0 = 0，然后发出 M5EXIT##RISCV。

## 指令级结构要求

1. 阶段 A：控制流与目标地址链准备。
2. 阶段 B：分支目标非对齐候选窗口。
3. 阶段 C：取指访问受限窗口。
4. 阶段 D：恢复与重试窗口。
5. 阶段 E：回汇收敛与退出。

## 数据编排要求

1. 维护 `BR_TARGET_SET = {ALIGNED, MISALIGNED}`。
2. 维护 `IFETCH_PAGE_SET = {EXEC_OK, EXEC_LIMIT}`。
3. 至少一组场景只改变目标对齐属性，不改变其余控制流模板。
4. 至少一组场景只改变取指页属性，不改变目标构造链。

## 通过判定

1. trace 可区分非对齐候选和取指故障候选窗口。
2. 两类候选都出现并具有恢复后的后继块。
3. 执行最终可收敛退出。

## 实现边界

- 仅限 RV64 单线程。
- 不引入向量主线。
- 重点是取指路径异常细节。

## 交付物

- 一个完整的 FORCE-RISCV Python 脚本。
- 一段简短说明，解释两类取指异常候选的对照设计与恢复逻辑。
